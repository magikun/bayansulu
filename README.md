# 🐪 Bayan Sulu Kids (Бота Балалар) — MVP Mobile Web App

### Проект для Bayan Sulu Hackathon 2026 🏆
**Направление:** Продуктовая разработка для бренда «Баян Сулу» (Линейка детских сладостей «Бота»)
**Дедлайн:** 6 мая 2026 года (23:59)

Bayan Sulu Kids — это мобильное адаптивное веб-приложение, созданное как интерактивный цифровой мост между физической продукцией бренда «Бота» (через QR-коды на упаковках конфет) и образовательной геймификацией для детей 7–11 лет.

---

## 🎯 Бизнес-ценность проекта (Проблема и Решение)

* **Проблема:** У бренда нет прямого цифрового канала коммуникации с потребителем. Мамы (35-40 лет) испытывают легкое чувство вины, покупая детям сладости («сладкое — вредно»).
* **Решение:** Приложение «Бота Балалар» превращает покупку конфет в полезное образовательное приключение по Казахстану. 
  * Мама видит пользу: ребёнок решает математические примеры, изучает историю Казахстана и строит юрты.
  * Ребёнок вовлечен: зарабатывает виртуальные «Ботакоины» 🪙 за знания и обменивает их на **реальные скидки/купоны** для следующей покупки сладостей в фирменных магазинах.
  * Бизнес получает: взрывной рост Retention (возвращаемости), LTV клиента и прямую связь с семьями.

## 🌟 Что было реализовано в рамках хакатона (MVP-Функционал)

Проект представляет собой полностью готовый к демонстрации, рабочий прототип мобильного приложения (адаптивный веб), включающий в себя:

1. **Интерактивный онбординг от КамБота:** Приветствие от талисмана бренда верблюжонка КамБота, выбор возраста (7–11 лет), ввод имени и выбор одного из 6 аватаров-символов Казахстана с начислением стартовых Ботакоинов.
2. **Стилизованная Карта Путешествий:** Анимированная векторная SVG-карта Казахстана с облаками, свечением и 5 географическими локациями, каждая из которых содержит свою мини-игру.
3. **5 Полноценных мини-игр (вместо минимальных 3 по ТЗ):**
   * **«Собери сладости» (Алматы) — Memory:** Карточки на развитие зрительной памяти с казахскими символами и сладостями «Баян Сулу».
   * **«Сборка Юрты» (Туркестан) — Puzzle:** Логическая и культурная игра по пошаговой сборке традиционной казахской юрты (Шаңырақ, Кереге, есік) и её декорированию.
   * **«Счёт с Ботой» (Степь) — Math Battle:** Арифметическая битва с монстрами-сладостями, развивающая навыки быстрого счёта.
   * **«Викторина» (Астана) — Quiz:** 15 образовательных вопросов по культуре, природе и географии Казахстана.
   * **«Camel Runner» (Чарын) — Action:** Раннер на координацию движений, где КамБот собирает монеты в Чарынском каньоне.
4. **Ежедневный сундук наград & Стрик:** Анимированный сундук с рандомными бонусами опыта и монет, а также трекер посещаемости за неделю.
5. **🛍️ Магазин Призов (Ботакоины ➡️ Реальные Скидки):** Полноценная витрина сладостей «Бота». Ребёнок обменивает заработанные Ботакоины на купоны. Приложение генерирует **визуальный штрих-код** и уникальный промокод для предъявления на кассах магазинов.
6. **Кабинет Родителя (Parent Mode):** Защищённый PIN-кодом кабинет с ограничением времени сессии (интерфейсный таймер) и детальной образовательной аналитикой успехов ребёнка.
7. **🔊 Умный Голосовой Помощник (TTS):** Интегрированный синтезатор речи (Web Speech API). При нажатии на облачка КамБот озвучивает подсказки приятным детским тембром на русском или казахском языках.
8. **🌐 Двуязычный интерфейс:** Полная мгновенная локализация интерфейса на казахский (`kk`) и русский (`ru`) языки через переключатель в шапке.

## 🛠️ Технический Стек & Архитектура

* **Фронтенд:** React 19 (с поддержкой `verbatimModuleSyntax`), TypeScript 6, Vite 8
* **Стилизация:** Vanilla Tailwind CSS, CSS-переменные, кастомные Keyframes-анимации в `globals.css` (плавание облаков, мерцание, сбор монет)
* **Анимации:** Framer Motion (плавные переходы страниц, частицы, всплытие модальных окон)
* **Управление состоянием & БД:** Zustand + `persist` middleware.
  * **Резильентная Архитектура:** Все данные о монетах, купленных купонах, уровнях и бейджах сохраняются в `localStorage`. Приложение работает **полностью оффлайн**. Оно никогда не зависнет и не упадёт из-за отсутствия интернета или падения бэкенда во время презентации жюри.

## 📦 Быстрый запуск и Тестирование локально

Убедитесь, что у вас установлен Node.js (рекомендуется v18+) и Python (v3.10+).

### 1. Запуск Фронтенда (React + Vite)
```bash
# Установите зависимости фронтенда
npm install --prefix front

# Запустите сервер разработки
npm run dev --prefix front
```
Перейдите по адресу [http://localhost:5173](http://localhost:5173). Переключите отображение в браузере в **мобильный режим** (разрешение мобильного телефона, например, iPhone 14), так как интерфейс оптимизирован под мобильные экраны (max-width: 430px).

### 2. Запуск Бэкенда (Python + FastAPI)
Бэкенд отвечает за облачную синхронизацию профилей игроков, глобальную таблицу лидеров и верификацию купонов.
```bash
# Перейдите в папку бэкенда
cd back

# Создайте виртуальное окружение (рекомендуется)
python -m venv venv
source venv/bin/activate  # Для Windows: venv\Scripts\activate

# Установите зависимости бэкенда
pip install -r requirements.txt

# Запустите FastAPI сервер
uvicorn main:app --reload --port 8000
```
Бэкенд запустится на [http://localhost:8000](http://localhost:8000). Интерактивная документация API (Swagger Docs) доступна по адресу [http://localhost:8000/docs](http://localhost:8000/docs).

## 🔄 Как работает автоматическая фоновая синхронизация?

Наш проект спроектирован по принципу **Offline-First с автосинхронизацией**:
1. **Режим Offline:** Если FastAPI-сервер не запущен, приложение работает полностью локально, сохраняя баланс Ботакоинов, опыт и полученные бейджи в `localStorage` вашего браузера. Ничего не теряется и не виснет.
2. **Режим Online:** Как только вы запускаете FastAPI сервер (на порту `8000` локально или прописываете URL в настройках Vercel), фронтенд **автоматически обнаруживает сервер и в фоновом режиме бесшовно синхронизирует** профиль игрока при любом ключевом действии (прохождение игры, трата монет, получение бейджа).
3. **Облачное сохранение:** База данных на сервере обновляется мгновенно, позволяя отслеживать успехи ребёнка с любого устройства и формировать глобальный лидерборд!

## 🧪 Запуск автоматических проверок (Build & Type Check)

Для проверки сборки и типов на отсутствие ошибок выполните команду:
```bash
npm run build --prefix front
```
Сборка скомпилируется в папку `front/dist/` с результатом **`Exit code: 0`** (0 ошибок).

## ☁️ Инструкция по деплою на Vercel

Чтобы быстро развернуть приложение на Vercel для демонстрации жюри живой ссылки:

1. **Авторизуйтесь в Vercel CLI:**

   ```bash
   npx vercel --cwd front --prod --yes
   ```
3. Vercel выдаст вам публичную живую ссылку вида `https://front-eta-ashy.vercel.app`. Вы сможете запустить её прямо на мобильном телефоне!
